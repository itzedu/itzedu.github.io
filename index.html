Installation I

Installing Ruby and Ruby on Rails is something that all developers in any operating system struggle with. Because of the inner workings of each OS, there are many tools for Linux, Mac OS X and Windows that people have created to help with the installation of RoR. However, configurations and dependencies are always going to be different depending on the OS. This causes bugs where a developer in one OS will get a an error that others in another OS would not. These types of inconsistencies hurt productivity while developing in any language or framework.

Enters Vagrant

Why Vagrant? Vagrant is a tool to create configurable, denpendable, and consistent development environments.

... so whether you are working on Linux, Mac OS X, or Windows, all your team members are running code in the same environment, against the same dependencies, all configured the same way. Say goodbye to "works on my machine" bugs. - Vagrant

Does't this sound awesome? Everyone in this track will be running the same exact environment. Therefore, if someone runs into a configuration or dependency error, the solution can be quickly shared to everyone! This means that almost all of our bugs will be caused by the developer themselves, instead of obscure environment errors. In this chapter, we will set up Vagrant, install RoR in our virtual machines, and set up a static IP address for us to interact with our RoR application thorugh a browser.

Set Up Vagrant

Vagrant allows users to easily create virtual machines. For those unfamiliar with VMs, they are simply an emulation of a computer system. For example, you can have a Mac OS X computer running a Windows virtual machine. To spin up virtual machines, Vagrant uses VirtualBox, a free virtualization software.

  1. Download VirtualBox for your operating system.
  2. Download Vagrant 1.8.6 for your operating system.

Getting Started

The first vagrant command that we are going to learn is vagrant init. This built in command will create something called a Vagrantfile. This file is what Vagrant uses to configure our virtual machines. Lets create a directory and run the init command.

$ mdkir first_vagrant_box
$ cd first_vagrant_box
$ vagrant init

Now you should have a Vagrantfile in your directory. Feel free to open it in your text editor but do not change anything yet.

Boxes

Instead of installing virtual machines from scratch every time, Vagrant uses a base image system to quickly make copies of any virtual machine. These images are called Vagrant "boxes". Vagrant has a whole catalog of public boxes that we can download. You can visit at HashiCorp's Atlas catalog. Let's add the ubuntu/trusty64 box to our computer.

$ vagrant box add ubuntu/trusty64

This command downloads that box from the catalog and saves it globally. To see a list of your boxes you can run

$ vagrant box list

Next, we will configure our Vagrantfile to use the box that we just downloaded. Open the Vagrantfile and let's configure our virtual machine to boot up with our ubuntu box.

Vagrant.configure("2") do |config|
  config.vm.box = "ubuntu/trusty64"
end

Vagrant Up

So far, we have done a couple things.

  1. Downloaded a ubuntu image.
  2. Initialized a directory with a Vagrantfile that uses that image.

Next, we have to actually spin up that image into a running virtual machine. In your console, run:

$ vagrant up

Once the VM is up and running, we can ssh into it.

$ vagrant ssh

This command allows you to ssh into your virtual machine and interact with it. Soon, we will install Ruby and Ruby On Rails into our machines!

Synced Folder

Everything is great so far, but how do we actually develop with our VM. Unlesss you are an expert in terminal editors such as VIM or Nano, writing code in your VMs does not seems like fun. So, Vagrant comes with a synced folder functionality. While you are ssh'ed into your VM, run:

vagrant@vagrant-ubuntu-trusty-64:~$ /vagrant
vagrant@vagrant-ubuntu-trusty-64:~$ ls
Vagrantfile

This /vagrant directory in your VM is automatically synced with the host machine's directory where vagrant init was executed. This means, if we open another terminal window and create a new file in host's firstvagrantbox directory, we will see that file in the VM /vagrant directory

$ cd first_vagrant_box
$ touch is_this_file_synced

Moment of truth.

vagrant@vagrant-ubuntu-trusty-64:~$ /vagrant
vagrant@vagrant-ubuntu-trusty-64:~$ ls
is_this_file_synced  Vagrantfile
